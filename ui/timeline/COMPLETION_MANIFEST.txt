================================================================================
KEY DESIGN DECISIONS
================================================================================

1. QWidget + QPainter vs QGraphicsView
   - Direct QPainter control for batch rendering
   - 7-pass pipeline: background → grid → events → playhead
   - Pixel-snapped grid (no anti-aliasing)
   - Anti-aliased event edges (curves)

2. Shared State Pattern
   - Single TimelineState object
   - All three widgets reference same state
   - Updates via direct mutation
   - Synchronization via canvas.update() call

3. Aggressive Culling
   - visible_events() uses O(log n) bisect
   - Filters by time window first
   - Then filters by layer
   - Handles 500 events at 60fps

4. Adaptive Grid
   - Tick interval chosen based on zoom level
   - Major/minor tick separation
   - Time labels formatted contextually (M:SS or M:SS.ff)

5. Playhead Advancement
   - Real-time progression at 1.0x speed
   - QTimer at 60fps (~16ms interval)
   - Loops back to 0.0 at end of content
   - Can be set via ruler click

6. Model Isolation
   - model.py has ZERO Qt dependencies
   - Can be used in CLI, testing, serialization
   - Dataclasses for clean, validated structures
