## Web Documentation Interface

**Open `docs/index.html` in your browser** for an interactive documentation interface with:
- Complete block reference (17+ block types)
- Architecture overview
- Feature module documentation
- Search functionality
- Organized navigation

The web interface automatically loads documentation data from `docs.json` (generated by `scripts/generate_docs.py`).

## Documentation Structure

EchoZero uses **co-located documentation** where docs live alongside code:

### Feature Documentation
Each feature module has its own README:
- `src/features/blocks/README.md` - Block entities and services
- `src/features/connections/README.md` - Connection management
- `src/features/execution/README.md` - Execution engine
- `src/features/projects/README.md` - Project management
- `src/features/setlists/README.md` - Setlist batch processing
- `src/features/show_manager/README.md` - Show Manager and MA3 sync
- `src/features/ma3/README.md` - GrandMA3 integration

### Application Documentation
- `src/application/blocks/README.md` - Block processors (execution logic)
- `src/application/commands/README.md` - Command system
- `src/application/settings/README.md` - Settings system

### UI Documentation
- `ui/qt_gui/README.md` - Qt GUI overview
- `ui/qt_gui/widgets/timeline/README.md` - Timeline widget

## Generating Documentation

To update the web documentation with current codebase state:

```bash
python3 scripts/generate_docs.py
```

This generates `docs/docs.json` with:
- All available block types and their metadata
- Feature module information
- Current application state

---

## Documentation Standards

- **Co-located** - Docs live with the code they document
- **Up-to-date** - Documentation reflects current implementation
- **Searchable** - Web interface provides search functionality
- **Organized** - Clear structure and navigation
