GetChildClass(light_userdata:handle): string:class_name
 -- Returns the class name of children that can be created for a given parent object.
 -- Use this to discover what child class types a parent object can have without iterating existing children.
 -- 
 -- LIMITATION: Returns only a SINGLE child class (typically the default/primary child class).
 --             If a parent can create multiple child types, this only returns one of them.
 -- 
 -- Examples:
 --   - A Track object returns "TimeRange" (tracks contain TimeRange children)
 --   - A TimeRange object returns "CmdSubTrack" or "FaderSubTrack" (time ranges contain sub-track children)
 -- 
 -- Use cases:
 --   1. Validate structure before calling Acquire() to create children
 --   2. Understand object hierarchy without examining existing children
 --   3. Debug why Acquire() might fail (e.g., if parent doesn't support the requested child class)
 -- 
 -- Note: This differs from GetClass() which returns the class of an existing object.
 --       GetChildClass() tells you what type of children a parent can create.
 -- 
 -- WORKAROUND to discover ALL possible child types:
 --   Since GetChildClass() returns only one type, use these methods to find all possible child classes:
 --   1. GetChildClass() - get the default/primary child class
 --   2. Children() + GetClass() - inspect existing children to see what types actually exist
 --   3. Acquire(class_name) - try creating with different class names (use ClassExists() first to validate)
 --   4. IsClassDerivedFrom() - check if child classes are derived from a base class
